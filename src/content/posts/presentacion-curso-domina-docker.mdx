"En mi máquina funciona."

Si alguna vez has pronunciado esa frase (o la has oído de un compañero de equipo mientras todos os mirabais con esa mezcla de vergüenza ajena y resignación), este curso es exactamente lo que necesitas. Docker no es solo una herramienta: **es la solución definitiva al problema más clásico del desarrollo de software**.

Bienvenido al curso **Domina Docker desde cero**: un curso completo y gratuito que te llevará desde no saber qué es un contenedor hasta desplegar aplicaciones en producción con seguridad, eficiencia y las mejores prácticas del sector.

### ¿Por qué Docker en 2026?

Hace diez años, desplegar una aplicación significaba rezar para que el servidor de producción tuviese exactamente la misma versión de Python, las mismas librerías del sistema y la misma configuración que tu portátil. Spoiler: nunca la tenía.

Docker resolvió esto de forma elegante: **empaqueta tu aplicación junto con todo lo que necesita para funcionar** — el runtime, las dependencias, la configuración — en un contenedor portátil que se ejecuta de forma idéntica en cualquier entorno. Tu portátil, el servidor de tu colega, el CI/CD, producción en AWS: mismo contenedor, mismo comportamiento.

En 2026, Docker no es opcional. Es la base sobre la que se construye todo lo demás: Kubernetes, microservicios, CI/CD moderno, cloud-native... Si quieres trabajar como desarrollador backend, DevOps o SRE, necesitas entender contenedores. Y Docker es el punto de entrada.

### ¿Qué hace diferente a este curso?

La mayoría de cursos de Docker fallan de la misma forma: te enseñan a ejecutar `docker run hello-world` y a escribir un Dockerfile básico, y luego te abandonan a tu suerte cuando intentas hacer algo real.

**Este curso es diferente:**

- **El "por qué" antes del "cómo"**: Empezamos explicando qué problema resuelven los contenedores y por qué surgieron. Si entiendes el problema, los comandos tienen sentido.
- **Seguridad desde el día uno**: Aprenderás a no ejecutar contenedores como root desde la primera lección. No como un "tip avanzado" al final: como la forma correcta de hacerlo.
- **Malas prácticas explícitas**: Te enseñaré qué NO hacer y por qué. Imágenes de 2GB, secretos hardcodeados en el Dockerfile, contenedores que escriben datos sin volúmenes... los errores más comunes, explicados para que no los cometas.
- **Escenarios reales**: Nada de ejemplos de juguete. Aprenderás con aplicaciones multi-contenedor, bases de datos, reverse proxies, y pipelines de CI/CD.
- **Docker moderno**: Docker 25+, BuildKit, multi-platform builds. No el Docker de 2018.

### ¿Necesito conocimientos previos?

Necesitas saber usar una terminal básica — navegar directorios, ejecutar comandos, eso es todo. No necesitas saber programar para entender Docker (aunque si ya programas, lo aprovecharás más rápido).

Lo que sí ayuda tener claro antes de empezar:

- Saber qué es un servidor y para qué sirve
- Entender que las aplicaciones tienen dependencias (librerías, runtimes, etc.)
- Curiosidad por entender cómo funciona el software que usas

Si ya tienes experiencia con Linux y la línea de comandos, vas a volar.

### ¿Listo para acabar con "en mi máquina funciona"?

Docker cambia fundamentalmente cómo piensas sobre el despliegue de software. Una vez que entiendes contenedores, te preguntas cómo sobreviviste sin ellos.

Este curso está diseñado para que al final no solo sepas usar Docker, sino que **entiendas por qué funciona así**. Esa comprensión es lo que distingue a quien ejecuta comandos de quien sabe lo que está haciendo.
